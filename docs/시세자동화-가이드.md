# 시세 자동 갱신 설정 가이드

공시지원금(통신사×요금제×기종)과 번쩍할인 시세를 **코드 수정 없이** 매일 자동으로 사이트에 반영하는 방법입니다.

```
[데이터 소스] → 매일 06:00 GitHub Actions가 수집 → auto-data.js 갱신 → 사이트 자동 배포
```

두 사이트(폰드림/폰마트)에 동시에 반영되며, 수동으로 즉시 갱신하고 싶을 땐
GitHub 저장소 → Actions → "Update price data daily" → **Run workflow** 버튼을 누르면 됩니다.

---

## 방식 1. 구글시트 연동 (추천 — 10분 설정, 확실하게 동작)

사장님(또는 직원)이 **구글시트에 숫자만 입력**하면 매일 사이트에 자동 반영됩니다.
스마트초이스(smartchoice.or.kr)에서 공시지원금을 확인해 시트에 붙여넣는 방식이라
완전 자동은 아니지만, 코드를 전혀 만지지 않아도 되고 즉시 운영 가능합니다.

### ① 시트 만들기

구글시트 하나에 탭 2개를 만듭니다.

**탭 1 — 공시지원금** (요금제 구간별로 한 줄씩):

| id | carrier | plan_monthly | official |
|---|---|---|---|
| s26-ultra | SKT | 125000 | 530000 |
| s26-ultra | SKT | 89000 | 400000 |
| s26-ultra | SKT | 49000 | 250000 |
| s26-ultra | KT | 130000 | 550000 |
| ... | | | |

**탭 2 — 총지원금(시세)** (유지조건 요금제 기준, 매일 바뀌는 값):

| id | carrier | mnp | chg |
|---|---|---|---|
| s26-ultra | SKT | 1150000 | 820000 |
| s26-ultra | KT | 1190000 | 850000 |
| ... | | | |

`id`는 사이트 기종 ID입니다 (`assets/js/data.js`의 각 기종 `id` 값과 동일하게).

### ② 시트 발행 (CSV)

각 탭마다: **파일 → 공유 → 웹에 게시** → 해당 탭 선택 → 형식 **CSV** → 게시 → URL 복사

### ③ GitHub에 등록

저장소 → Settings → Secrets and variables → **Actions** → New repository secret:

- `SHEET_OFFICIAL_CSV` = 공시지원금 탭 CSV URL
- `SHEET_SUPPORT_CSV` = 총지원금 탭 CSV URL

끝. 다음 새벽부터 자동 반영됩니다. (즉시 반영: Actions → Run workflow)

---

## 방식 2. 공공데이터포털 API (완전 자동 — 단, 확인 필요)

정부(과기정통부/KAIT)가 운영하는 스마트초이스의 단말기 지원금 데이터를
공공데이터포털(data.go.kr) Open API로 받아오는 방식입니다.

1. [data.go.kr](https://www.data.go.kr) 회원가입 → "단말기 지원금" 검색 → 활용 신청 (무료, 즉시~1일)
2. 발급받은 **일반 인증키**와 API **엔드포인트 URL**을 Secrets에 등록:
   - `DATA_GO_KR_KEY`
   - `DATA_GO_KR_ENDPOINT`
3. `scripts/update-prices.mjs`의 `mapDataGov()` 함수에서 API 응답 필드명을
   신청한 API 문서에 맞게 1회 수정 (이 부분은 API 승인 후 저에게 응답 예시를 주시면 맞춰드립니다)

> ⚠️ 솔직한 안내: 일부 한국 공공 사이트는 해외 IP(GitHub 서버 포함)를 차단하는 경우가 있어,
> API 승인 후 첫 실행에서 확인이 필요합니다. 차단될 경우 방식 1(구글시트)을 사용하세요.
> 두 방식이 모두 설정돼 있으면 API를 먼저 시도하고 실패 시 시트를 사용합니다.

---

## 동작 확인 방법

1. Actions 탭에서 "Update price data daily" 실행 로그 확인
2. 성공 시 `assets/js/auto-data.js`에 "시세 자동 갱신 (날짜)" 커밋이 생김
3. 사이트 새로고침 → 시세표/상세페이지 금액이 시트 값으로 변경됨

자동 데이터가 없거나 실패하면 사이트는 기존처럼 `data.js`의 수동 값을 그대로 사용하므로
사이트가 깨질 걱정은 없습니다.

## 참고: 예약(cron) 실행 조건

GitHub의 예약 실행은 **기본 브랜치(main)** 의 워크플로우만 작동합니다.
현재 사이트는 `claude/serene-wright-yyxaca` 브랜치에 있으므로:
- 지금 당장은 **Run workflow 수동 실행**으로 사용 (브랜치 선택 가능)
- 사이트를 main에 합치면(merge) 매일 자동 실행이 활성화됩니다
